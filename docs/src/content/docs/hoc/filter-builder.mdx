`FilterBuilder` renders rows from `fields` definitions. Each row picks a field, an operator (defaults depend on field `type`), and a value when the operator needs one.

## Usage

```tsx
import { FilterBuilder } from "shadcn-solid-components/hoc/filter-builder"

<FilterBuilder
  fields={[
    { key: "name", label: "Name", type: "text" },
    { key: "status", label: "Status", type: "select", options: [{ label: "Open", value: "open" }] },
  ]}
  value={rules()}
  onChange={setRules}
/>
```

`value` is a controlled array: adding a row, changing a field, or switching an operator will all
trigger `onChange`. `maxRules` can limit the number of conditions, and `select` fields must provide
`options`. The default operator set includes values such as `eq` and `in`.

## API

### `FilterBuilderProps`

Extends `div` props (without `onChange`).

- `fields` — `FilterFieldDefinition[]`
- `value` — `FilterRule[]`
- `onChange` — `(rules: FilterRule[]) => void`
- `maxRules`, `locale`

### `FilterFieldDefinition`

- `key`, `label`, `type` (`'text' | 'number' | 'select' | 'date'`)
- `operators` — optional override list
- `options` — for `select` type

### `FilterRule`

- `field`, `operator`, `value`

### `FilterOperator`, `FilterFieldType`, `FilterFieldOption`

See `src/hoc/filter-builder/index.tsx` for the operator union and defaults per type.
