`SettingsLayout` provides a full-page layout for settings screens. On desktop it renders a fixed
sidebar with navigation items; on mobile (below `lg` breakpoint) the sidebar slides in as a drawer.
Content is keyed by section ID and swapped when the user navigates between sections.

## Usage

```tsx
import { SettingsLayout } from "shadcn-solid-components/hoc/settings-layout"

<SettingsLayout
  sections={[
    { id: "profile", label: "Profile" },
    { id: "security", label: "Security" },
  ]}
  defaultActiveSection="profile"
>
  {{
    profile: <ProfileSettings />,
    security: <SecuritySettings />,
  }}
</SettingsLayout>
```

Children can be either a `Record<string, JSX.Element>` object keyed by section ID, or a render
function `(sectionId: string) => JSX.Element` for lazy rendering.

### Render function children

Use a render function for lazy content rendering:

```tsx
import { SettingsLayout } from "shadcn-solid-components/hoc/settings-layout"

<SettingsLayout sections={sections} defaultActiveSection="profile">
  {(sectionId) => {
    switch (sectionId) {
      case "profile":
        return <ProfileSettings />
      case "security":
        return <SecuritySettings />
      default:
        return <p>Unknown section</p>
    }
  }}
</SettingsLayout>
```

### Controlled active section

```tsx
import { createSignal } from "solid-js"
import { SettingsLayout } from "shadcn-solid-components/hoc/settings-layout"

export function ControlledSettings() {
  const [active, setActive] = createSignal("profile")

  return (
    <SettingsLayout
      sections={sections}
      activeSection={active()}
      onSectionChange={setActive}
    >
      {{
        profile: <ProfileSettings />,
        security: <SecuritySettings />,
      }}
    </SettingsLayout>
  )
}
```

### Custom sidebar width

```tsx
<SettingsLayout sections={sections} sidebarWidth="300px">
  {{ ... }}
</SettingsLayout>
```

### Hidden sidebar (content only)

```tsx
<SettingsLayout sections={sections} hideSidebar>
  {{ profile: <ProfileSettings /> }}
</SettingsLayout>
```

## API

### `SettingsLayoutProps`

Extends `div` props (without `children`).

- `sections` — `SettingsSection[]` — sidebar navigation items
- `activeSection` — `string` — controlled active section ID
- `defaultActiveSection` — `string` — default active section ID (uncontrolled)
- `onSectionChange` — `(sectionId: string) => void` — callback on navigation
- `children` — `Record<string, JSX.Element>` | `(sectionId: string) => JSX.Element` — content keyed by section ID
- `sidebarWidth` — `string` — custom sidebar width CSS value (default `256px` / `w-64`)
- `hideSidebar` — `boolean` — hide sidebar, content takes full width

### `SettingsSection`

- `id` — unique section identifier
- `label` — navigation item text
- `icon` — optional icon element

## Responsive behaviour

| Breakpoint | Sidebar      | Toggle button         |
| ---------- | ------------ | --------------------- |
| `< lg`     | Hidden (drawer) | Fixed top-left button |
| `>= lg`    | Fixed left   | Hidden                |

## i18n

Locale key: `SettingsLayout`. Overridable strings:

| Key              | Default          |
| ---------------- | ---------------- |
| `toggleSidebar`  | Toggle sidebar   |
