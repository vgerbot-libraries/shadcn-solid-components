Search combines an input-like control, optional icon and loading indicator, and a listbox for filtered options. It is generic over option and optional group types, matching Kobalte’s `Search` API.

## Usage

```tsx
import {
  Search,
  SearchContent,
  SearchControl,
  SearchIcon,
  SearchIndicator,
  SearchItem,
  SearchItemLabel,
  SearchLabel,
  SearchListbox,
  SearchPortal,
  SearchSection,
} from "shadcn-solid-components/components/search"
```

Compose `SearchControl` with `SearchIcon` / `SearchIndicator`, attach `SearchContent` for the popover layer, and render options inside `SearchListbox` using `SearchItem` and related parts. See `src/components/search/index.tsx` for the full part list (`SearchNoResult`, `SearchDescription`, and others).

## Exports

- `Search`, `SearchPortal`, `SearchIcon`, `SearchIndicator`, `SearchControl`, `SearchContent`, `SearchNoResult`, `SearchItem`, `SearchLabel`, `SearchDescription`, `SearchItemDescription`, `SearchItemLabel`, `SearchListbox`, `SearchSection`
- Types: `SearchProps`, `SearchIconProps`, `SearchIndicatorProps`, `SearchControlProps`, `SearchContentProps`, and matching `*Props` types for other parts
