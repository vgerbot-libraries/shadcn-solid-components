`Stepper` manages the active step (controlled via `activeStep` / `onActiveStepChange` or uncontrolled). Each step supplies `content` and an optional `validate` callback run before advancing. Child content can call `useStepper()` for `next`, `prev`, `goTo`, and state accessors.

## Usage

```tsx
import { Stepper } from "shadcn-solid-components/hoc/stepper"

<Stepper
  steps={[
    { label: "Account", content: <AccountFields /> },
    { label: "Review", content: <Review />, validate: () => formValid() },
  ]}
  onComplete={() => submit()}
/>
```

### Custom navigation with `useStepper`

```tsx
import { Button } from "shadcn-solid-components/components/button"
import { Stepper, useStepper } from "shadcn-solid-components/hoc/stepper"

function StepContent() {
  const api = useStepper()
  return (
    <div class="space-y-3">
      <p class="text-muted-foreground text-sm">Step {api.activeStep() + 1} of {api.totalSteps}</p>
      <div class="flex gap-2">
        <Button disabled={api.isFirst()} type="button" variant="outline" onClick={() => api.prev()}>
          Back
        </Button>
        <Button type="button" onClick={() => void api.next()}>
          {api.isLast() ? "Finish" : "Continue"}
        </Button>
      </div>
    </div>
  )
}

<Stepper
  showNavigation={false}
  steps={[
    { label: "Account", content: <StepContent />, validate: () => true },

  ]}
  onComplete={() => console.info("done")}
/>
```

### `validate` blocks moving forward

If the current step's `validate` returns `false` or resolves to `false`, `api.next()` will not
advance the step index. The built-in "Next" button follows the same rule and stays blocked until
validation passes.

```tsx
import { createSignal, type Accessor } from "solid-js"
import { Button } from "shadcn-solid-components/components/button"
import { Stepper, useStepper } from "shadcn-solid-components/hoc/stepper"

function TermsStep(props: { agreed: Accessor<boolean>; setAgreed: (v: boolean) => void }) {
  const api = useStepper()
  return (
    <div class="space-y-3">
      <label class="flex items-center gap-2 text-sm">
        <input
          type="checkbox"
          checked={props.agreed()}
          onChange={e => props.setAgreed(e.currentTarget.checked)}
        />
        I agree to the terms
      </label>
      <Button type="button" onClick={() => void api.next()}>
        Continue
      </Button>
    </div>
  )
}

const [agreed, setAgreed] = createSignal(false)

<Stepper
  showNavigation={false}
  steps={[
    {
      label: "Terms",
      content: <TermsStep agreed={agreed} setAgreed={setAgreed} />,
      validate: () => agreed(),
    },
    { label: "Done", content: <p>All steps completed.</p> },
  ]}
/>
```

## Exports

| Name | Description |
| --- | --- |
| `Stepper` | Wizard component. |
| `StepperProps` | `steps`, optional `activeStep`, `onActiveStepChange`, `variant` (`'default' \| 'circle' \| 'dot'`), `orientation`, `clickable` (defaults to `true`), `showNavigation` (defaults to `true`), `onComplete`, `locale`, plus `div` props. |
| `StepItem` | `label`, optional `description`, `icon`, `content`, optional `validate`. |
| `StepperVariant` | `'default' \| 'circle' \| 'dot'`. |
| `StepperOrientation` | `'horizontal' \| 'vertical'`. |
| `useStepper` | Returns `StepperApi` when used inside `<Stepper>`. |
| `StepperApi` | `activeStep`, `isFirst`, `isLast` are accessors; `totalSteps` is a number; `next` is async and runs the current stepÔÇÖs `validate` before advancing; `prev` goes back; `goTo(step)` only updates when `step` is in range and not greater than the current step (for example when jumping to a completed step with `clickable`). |
| `enLocale`, `zhCNLocale`, `zhTWLocale`, `jaLocale` | `StepperLocale` presets for merging or reference. |
