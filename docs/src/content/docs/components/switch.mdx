The switch composes Kobalte’s switch primitive: `Switch` is the root; `SwitchControl` and `SwitchThumb` render the track and knob; `SwitchInput` is the accessible input. Optional `SwitchLabel`, `SwitchDescription`, and `SwitchErrorMessage` match the text-field patterns.

## Installation

Install `shadcn-solid-components` and satisfy peer dependencies (`solid-js`, Tailwind CSS v4, `tw-animate-css`).

## Usage

```tsx
import {
  Switch,
  SwitchControl,
  SwitchDescription,
  SwitchErrorMessage,
  SwitchInput,
  SwitchLabel,
  SwitchThumb,
} from "shadcn-solid-components/components/switch"
```

```tsx
<Switch>
  <SwitchLabel>Airplane mode</SwitchLabel>
  <SwitchControl>
    <SwitchThumb />
  </SwitchControl>
  <SwitchInput />
</Switch>
```

Pass `disabled` on the root `Switch` to prevent interaction and dim the control. Keep `SwitchInput` in the tree so the checked state stays exposed to assistive tech and forms.

Use `checked` / `defaultChecked` and `onChange` when you need controlled or uncontrolled state like other form controls.

### Disabled

When `disabled` is set, the switch does not receive focus for toggling and should reflect that state visually (handled by the component styles).
