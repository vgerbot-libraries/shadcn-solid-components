The text field composes Kobalte’s text-field primitive: a root `TextField` groups `TextFieldLabel`, `TextFieldInput` or `TextFieldTextArea`, optional `TextFieldDescription`, and `TextFieldErrorMessage`. Styling integrates with the library theme helpers for input radius and focus rings.

## Installation

Install `shadcn-solid-components` and satisfy peer dependencies (`solid-js`, Tailwind CSS v4, `tw-animate-css`).

## Usage

```tsx
import {
  TextField,
  TextFieldDescription,
  TextFieldErrorMessage,
  TextFieldInput,
  TextFieldLabel,
  TextFieldTextArea,
} from "shadcn-solid-components/components/text-field"
```

Wire validation the same way as Kobalte (for example `validationState` and error messages). `TextFieldErrorMessage` accepts an optional `errors` array and renders a default list when `children` are not provided.

## Exports

- `TextField`, `TextFieldInput`, `TextFieldTextArea`, `TextFieldLabel`, `TextFieldDescription`, `TextFieldErrorMessage`
- Types: `TextFieldProps`, `TextFieldInputProps`, `TextFieldTextAreaProps`, `TextFieldLabelProps`, `TextFieldDescriptionProps`, `TextFieldErrorMessageProps`

### Error

<ComponentPreview name="Text Field Error Demo">
  <div class="mb-4 w-full">
    <TextFieldErrorDemo />
  </div>
  ```tsx
  import { createSignal } from "solid-js"

import {
  TextField,
  TextFieldErrorMessage,
  TextFieldInput,
  TextFieldLabel,
} from "shadcn-solid-components/components/text-field"

const TextFieldErrorDemo = () => {
  const [value, setValue] = createSignal("Orange")

  return (
    <TextField
      class="max-w-xs"
      value={value()}
      onChange={setValue}
      validationState={value() !== "Apple" ? "invalid" : "valid"}
    >
      <TextFieldLabel>Favorite fruit</TextFieldLabel>
      <TextFieldInput type="text" />
      <TextFieldErrorMessage>Hmm, I prefer apples.</TextFieldErrorMessage>
    </TextField>
  )
}

export default TextFieldErrorDemo
  ```
</ComponentPreview>
