Use `Alert` for inline, persistent feedback inside a page or section. Combine it with `AlertTitle` and `AlertDescription` when you need more structure than plain text.

## Installation

Install the package, then make sure the required peer dependencies are already configured in your app.

```bash
pnpm add shadcn-solid-components
```

Peer dependencies: `solid-js`, Tailwind CSS v4, and `tw-animate-css`.

## Usage

```tsx
import { Alert, AlertDescription, AlertTitle } from "shadcn-solid-components/components/alert"
```

```tsx
<Alert as="div">
  <AlertTitle>Heads up!</AlertTitle>
  <AlertDescription>
    Your trial ends in 3 days.
  </AlertDescription>
</Alert>
```

## Accessibility And Semantics

`Alert` is built on Kobalte's alert primitive and defaults to rendering as a `button`.

Use `as="div"` when the alert is informational and should not be interactive. Keep the default element, or render another interactive element, only when the entire alert is meant to be clicked.

## Variants

- `default`: General information, neutral status, or success-style messaging.
- `destructive`: Error, warning, or destructive state that needs stronger emphasis.

## Custom Styling

Import `alertVariants` when you want to reuse the same visual treatment on a custom wrapper or extend the built-in variants.

```tsx
import { alertVariants } from "shadcn-solid-components/components/alert"
```
