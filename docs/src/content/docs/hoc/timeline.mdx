`Timeline` renders a vertical list of events connected by a line. Each event is a **node** (dot or custom icon) paired with a **content block** (title, description, time, and optional rich content).

Three layout modes are available:

- **`left`** (default) — all events aligned to the left.
- **`right`** — all events aligned to the right.
- **`alternate`** — events alternate left and right on medium+ screens; collapses to left-aligned on mobile.

When `pending` is `true`, a pulsing node and a localized "Loading…" label are appended after the last item, useful for streams that are still in progress.

## Usage

```tsx
import { Timeline } from "shadcn-solid-components/hoc/timeline"

<Timeline
  mode="left"
  items={[
    { title: "Created", time: "10:00", description: "Order placed." },
    { title: "Shipped", color: "text-green-500" },
  ]}
/>
```

## API

### TimelineProps

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `items` | `TimelineItem[]` | — | The events to display. **Required.** |
| `mode` | `'left' \| 'right' \| 'alternate'` | `'left'` | Layout mode. |
| `pending` | `boolean` | `false` | Show a pulsing indicator after the last item. |
| `locale` | `Partial<TimelineLocale>` | — | Override the pending label text. |
| `class` | `string` | — | Additional CSS classes on the root element. |

### TimelineItem

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `title` | `string` | — | Primary heading for the event. **Required.** |
| `description` | `string \| JSX.Element` | — | Secondary text below the title. |
| `time` | `string` | — | Timestamp or date displayed beside the title. |
| `icon` | `JSX.Element` | — | Custom node icon. Defaults to a coloured dot. |
| `color` | `string` | — | Tailwind text-colour class for the node (e.g. `'text-blue-500'`). |
| `content` | `JSX.Element` | — | Extra content rendered below title/description. |

### TimelineLocale

| Key | Type | Default (`en-US`) |
| --- | --- | --- |
| `pending` | `string` | `'Loading...'` |

Override via the `locale` prop or globally through `ConfigProvider`.
