`ModeToggleDropdown` uses Kobalte `useColorMode` and the dropdown menu primitives. The trigger shows sun/moon icons with a cross-fade; menu items call `setColorMode` with `'light'`, `'dark'`, or `'system'`.

## Usage

```tsx
import { ModeToggleDropdown } from "shadcn-solid-components/hoc/mode-toggle-dropdown"

<ModeToggleDropdown />
```

Optional `trigger` spreads onto the `Button` trigger (e.g. `class`).

### Compact trigger and custom labels

```tsx
import { ModeToggleDropdown } from "shadcn-solid-components/hoc/mode-toggle-dropdown"

<ModeToggleDropdown
  trigger={{ class: "size-9 px-0" }}
  locale={{ light: "Light", dark: "Dark", system: "System" }}
/>
```

### Merging labels with `ConfigProvider`

The dropdown labels are first merged with `locale.ModeToggleDropdown` from `ConfigProvider`, and
then overridden by the component's own `locale` prop. This works well when you wrap the app root in
a global locale pack:

```tsx
import { ConfigProvider } from "shadcn-solid-components/components/config-provider"
import { ModeToggleDropdown } from "shadcn-solid-components/hoc/mode-toggle-dropdown"
import { zhCN } from "shadcn-solid-components/i18n/locales/zh-CN"

export function AppChrome() {
  return (
    <ConfigProvider locale={zhCN}>
      <ModeToggleDropdown />
    </ConfigProvider>
  )
}
```

## API

### `ModeToggleDropdownProps`

- `trigger` — props for the `Button` trigger (merged with defaults)
- `locale` — partial strings for light/dark/system labels and the sr-only toggle text
