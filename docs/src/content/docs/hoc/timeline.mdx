`Timeline` maps `items` to nodes connected by a vertical line. In `alternate` mode, content switches sides on medium screens and falls back to a single column on small viewports. When `pending` is `true`, a pulsing node and localized pending label (non-`alternate` layouts) are shown after the last item.

## Usage

```tsx
import { Timeline } from "shadcn-solid-components/hoc/timeline"

<Timeline
  mode="left"
  items={[
    { title: "Created", time: "10:00", description: "Order placed." },
    { title: "Shipped", color: "text-green-500" },
  ]}
/>
```

### Pending tail state

```tsx
import { Timeline } from "shadcn-solid-components/hoc/timeline"

<Timeline
  mode="left"
  pending
  items={[{ title: "Queued", description: "Job is waiting." }]}
/>
```

## Exports

| Name | Description |
| --- | --- |
| `Timeline` | Timeline component. |
| `TimelineProps` | `items`, optional `mode` (`'left' \| 'right' \| 'alternate'`), `pending`, `locale`, plus `div` props. |
| `TimelineItem` | `title`, optional `description` (`string` \| JSX), `time`, `icon`, `color` (Tailwind text class), `content`. |
| `TimelineMode` | Layout mode union type. |
